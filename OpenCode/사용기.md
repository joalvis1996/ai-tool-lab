## OpenCode란

OpenCode는 터미널 환경에서 작동하는 오픈소스 AI 코딩 에이전트로, 개발자들이 명령줄 인터페이스에서 직접 AI의 도움을 받아 코드를 작성하고 관리할 수 있도록 설계되었습니다. 기존의 웹 기반 AI 코딩 도구들과 달리, 개발자의 자연스러운 워크플로우인 터미널 환경에서 바로 작동한다는 점이 가장 큰 특징입니다.

oh-my-opencode는 OpenCode의 기능을 확장하고 다양한 SubAgent 및 도구들을 통합하여 제공하는 플러그인입니다.

## 왜 OpenCode가 뜨고 있는가?

### 1. 터미널 중심 개발 워크플로우의 혁신

대부분의 개발자들은 이미 터미널에서 작업하고 있습니다. Git 명령어, 패키지 설치, 서버 실행 등 모든 것이 터미널에서 이루어집니다. OpenCode는 이 자연스러운 워크플로우를 깨뜨리지 않고 AI를 통합했습니다. 별도의 웹 브라우저나 IDE 플러그인 없이, 개발자가 이미 사용하고 있는 터미널에서 바로 AI 코딩 어시스턴트를 사용할 수 있습니다.

### 2. 벤더 독립성과 모델 선택의 자유

OpenCode는 75개 이상의 LLM 제공업체를 지원하며, 로컬 모델도 사용할 수 있습니다. 이는 특정 AI 서비스에 종속되지 않고, 사용자가 원하는 모델을 자유롭게 선택할 수 있다는 의미입니다. OpenAI, Anthropic Claude, Google Gemini 등 어떤 모델이든 선택할 수 있으며, 프라이버시가 중요한 경우 로컬 모델을 사용할 수도 있습니다.

### 3. 완전한 오픈소스와 활발한 커뮤니티

OpenCode는 완전한 오픈소스 프로젝트로, 300명 이상의 기여자와 4,000건에 가까운 커밋을 통해 지속적으로 발전하고 있습니다. 이는 단순한 도구가 아닌, 커뮤니티가 함께 만들어가는 생태계라는 의미입니다.

### 4. 개발자 경험(DX)의 극대화

기존 AI 코딩 도구들이 웹 인터페이스나 IDE 플러그인으로 제공되면서 컨텍스트 전환이 발생했지만, OpenCode는 터미널에서 모든 것을 해결합니다. 파일 시스템 접근, 명령 실행, 코드 수정 등이 모두 하나의 환경에서 이루어집니다.

## OpenCode의 기술적 핵심 요소

### 1. 네이티브 터미널 사용자 인터페이스(TUI)

- **Bubble Tea 기반**: Go 언어의 Bubble Tea 프레임워크로 구축된 반응형이고 테마를 적용할 수 있는 터미널 UI
- **인터랙티브한 경험**: 웹 기반 도구와 유사한 사용자 경험을 터미널에서 제공
- **Vim 스타일 편집기**: 텍스트 입력 기능이 통합된 Vim 스타일의 편집기로, 개발자들에게 익숙한 환경 제공

### 2. 언어 서버 프로토콜(LSP) 통합

- **실시간 코드 인텔리전스**: 사용 중인 LLM에 맞는 LSP를 자동으로 로드하여 코드 작성 시 실시간 진단과 자동 완성 기능 제공
- **언어별 지원**: 다양한 프로그래밍 언어에 대한 LSP 지원으로 정확한 코드 분석과 제안 가능

### 3. 다중 세션 및 세션 관리

- **병렬 작업 지원**: 동일한 프로젝트에서 여러 에이전트를 병렬로 실행하여 복잡한 작업을 효율적으로 관리
- **세션 저장 및 관리**: SQLite 데이터베이스를 활용하여 대화와 세션을 저장하고 관리
- **공유 가능한 세션 링크**: 각 세션에 대한 링크를 생성하여 참조하거나 디버깅 목적으로 공유 가능

### 4. 강력한 도구 통합

- **파일 시스템 접근**: 프로젝트 내 파일을 직접 읽고 수정
- **명령 실행**: 터미널 명령어를 실행하여 빌드, 테스트, 실행 등의 작업 수행
- **파일 변경 추적**: 세션 중 파일 변경 사항을 추적하고 시각화하여 코드 수정 내역을 쉽게 파악
- **외부 편집기 지원**: 선호하는 외부 편집기를 열어 메시지를 작성할 수 있는 유연성

### 5. 프로젝트 컨텍스트 이해

- **프로젝트 구조 분석**: `/init` 명령어를 통해 프로젝트 구조를 분석하고 `AGENTS.md` 파일 생성
- **코딩 패턴 학습**: 프로젝트의 구조와 코딩 패턴을 이해하여 일관된 코드 생성
- **파일 참조 시스템**: `@` 키를 통한 파일 검색 및 참조로 컨텍스트 기반 답변 제공

### 6. 에이전트 아키텍처

- **build 에이전트**: 개발 작업을 위한 전체 액세스 에이전트로 코드 작성, 수정, 실행 등 모든 작업 수행
- **plan 에이전트**: 분석 및 코드 탐색을 위한 읽기 전용 에이전트로 코드 분석과 계획 수립에 특화
- **동적 에이전트 전환**: `Tab` 키를 통한 실시간 에이전트 전환으로 작업 유형에 맞는 도구 선택

### 7. 사용자 정의 명령 시스템

- **명명된 인자**: 여러 명명된 플레이스홀더를 사용하여 강력한 사용자 정의 명령 생성
- **재사용 가능한 워크플로우**: 반복적인 작업을 명령으로 저장하여 효율성 향상

### 8. 확장 가능한 플러그인 시스템

- **oh-my-opencode**: 다양한 SubAgent 및 도구들을 통합하는 플러그인 시스템
- **커뮤니티 기여**: 오픈소스 생태계를 통한 지속적인 기능 확장

## 왜 개발자들이 열광하는가?

1. **워크플로우의 자연스러운 통합**: 기존 개발 환경을 바꾸지 않고 AI를 활용할 수 있음
2. **완전한 제어권**: 오픈소스이므로 코드를 직접 확인하고 수정할 수 있음
3. **프라이버시**: 로컬 모델 사용 가능으로 코드가 외부로 전송되지 않음
4. **비용 효율성**: 다양한 모델 제공업체 중 선택 가능하여 비용 최적화 가능
5. **학습 곡선의 최소화**: 터미널을 사용하는 개발자라면 즉시 사용 가능
6. **강력한 기능**: LSP 통합, 다중 세션, 파일 추적 등 전문 개발 도구 수준의 기능 제공

## macOS에서 OpenCode 설치

### 방법 1: Homebrew를 이용한 설치 (권장)

```bash
brew install sst/tap/opencode
```

### 방법 2: 설치 스크립트를 이용한 설치

```bash
curl -fsSL https://opencode.ai/install | bash
```

### 방법 3: npm을 이용한 설치

```bash
npm install -g opencode-ai
```

## OpenCode 초기 설정

### 1. AI 모델 제공업체 설정

OpenCode는 다양한 AI 모델 제공업체를 지원합니다. 설치 후 다음 명령어로 설정을 진행합니다:

```bash
opencode config
```

또는 특정 제공업체를 직접 선택할 수 있습니다:

```bash
opencode --provider [provider_name]
```

지원되는 제공업체:
- OpenAI
- Anthropic (Claude)
- Google
- Azure
- 기타 다양한 LLM 제공업체

설정 과정에서 해당 제공업체의 API 키를 입력하여 인증을 완료합니다.

### 2. 프로젝트 초기화

프로젝트 디렉토리로 이동한 후 OpenCode를 실행합니다:

```bash
cd /path/to/your/project
opencode
```

프로젝트에서 처음 OpenCode를 사용하는 경우, `/init` 명령어를 실행하여 초기화를 진행합니다:

```bash
/init
```

이 명령어는 프로젝트 구조를 분석하고 `AGENTS.md` 파일을 생성합니다. 이 파일은 OpenCode가 프로젝트의 구조와 코딩 패턴을 이해하는 데 도움을 줍니다.

## OpenCode 기본 사용법

### 실행

터미널에서 `opencode` 명령어를 입력하여 OpenCode를 실행합니다:

```bash
opencode
```

### 주요 기능

1. **파일 참조 기능**
   - 프롬프트에서 `@` 키를 누르면 프로젝트 내 파일을 검색하여 프롬프트에 포함시킬 수 있습니다
   - 이를 통해 OpenCode가 해당 파일의 내용을 참고하여 답변을 제공합니다

2. **에이전트 전환**
   - OpenCode는 두 가지 내장 에이전트를 제공합니다:
     - **build**: 개발 작업을 위한 전체 액세스 에이전트
     - **plan**: 분석 및 코드 탐색을 위한 읽기 전용 에이전트
   - `Tab` 키를 사용하여 이들 에이전트 간에 전환할 수 있습니다

3. **자연어 명령**
   - 자연어로 명령을 입력하여 코드 생성, 리팩토링, 디버깅 등 다양한 개발 작업을 수행할 수 있습니다

## oh-my-opencode 설치

oh-my-opencode는 OpenCode의 기능을 확장하는 플러그인입니다.

### 설치 방법

**Bun을 이용한 설치:**

```bash
bunx oh-my-opencode install
```

**Node.js를 이용한 설치:**

```bash
npx oh-my-opencode install
```

설치 스크립트는:
- 사용자의 모델 구독 여부를 확인합니다
- `opencode.json` 플러그인 설정을 자동으로 구성합니다

## oh-my-opencode 사용법

설치가 완료되면 OpenCode 내에서 oh-my-opencode의 다양한 SubAgent 및 도구들을 활용할 수 있습니다. 이 플러그인을 통해 개발 작업을 더욱 효율적으로 수행할 수 있습니다.

## 활용 팁

1. **프로젝트 초기화는 필수**: `/init` 명령어를 실행하여 프로젝트 구조를 분석하면 더 정확한 코드 생성이 가능합니다

2. **파일 참조 활용**: `@` 키를 사용하여 관련 파일을 프롬프트에 포함시키면 컨텍스트를 이해한 더 나은 답변을 얻을 수 있습니다

3. **에이전트 선택**: 
   - 코드 작성이나 수정이 필요한 경우 → `build` 에이전트 사용
   - 코드 분석이나 탐색이 필요한 경우 → `plan` 에이전트 사용

4. **자연어 명령 구체화**: 구체적이고 명확한 명령을 입력할수록 더 정확한 결과를 얻을 수 있습니다
